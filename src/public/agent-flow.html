PIPELINE LAYER — NEW CROSS-CUTTING SERVICES
NEW SERVICE
⚙️ PipelineRunner
src/services/pipeline-runner.ts

Tracks each lead through sequential steps:
INTAKE → ENRICH → BRIEF → PROPOSAL → OUTREACH

States per lead:
🟢 WAITING — next step ready
🟡 RUNNING — step executing
🔴 BLOCKED — missing data / step failed
✅ DONE — all steps complete

Wired into:
• Agent-01 → start() on HOT/WARM leads
• Agent-18 → start() on email leads
• Agent-02 → advance() after enrichment
• Agent-03 → advance() or block() after brief

Persists to KB on every state change
Loads from KB at startup (survives restarts)
NEW UTIL
✅ DataValidator
src/utils/data-validator.ts

Checks required fields before each pipeline step:

validateForEnrichment(row)
→ needs: company name
→ warns: contact email, industry

validateForBrief(row)
→ needs: company, show, size, budget
→ warns: industry

validateForOutreach(row)
→ needs: company, email
→ warns: show, readiness score

Agent-03 calls block() if fields missing
